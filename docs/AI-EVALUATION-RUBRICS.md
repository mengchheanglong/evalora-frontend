# AI Evaluation Rubrics

AI evaluation in Evalora must be structured, evidence-based, and advisory.

## Global scoring scale

| Score | Meaning |
| --- | --- |
| 1 | Very weak |
| 2 | Weak |
| 3 | Acceptable |
| 4 | Good |
| 5 | Excellent |

## Required AI output shape

Each module evaluation should include:

- `score`: number from 1 to 5.
- `criteriaScores`: per-rubric breakdown.
- `summary`: short explanation.
- `strengths`: specific positive evidence.
- `improvementAreas`: specific gaps.
- `evidence`: quotes or references from candidate responses.
- `advisoryNotice`: statement that the result supports human review.

## Technical/coding dimensions

- Correctness.
- Problem solving.
- Code clarity.
- Debugging ability.
- Efficiency awareness.
- Explanation quality.

## Behavioral/work-style dimensions

- Collaboration.
- Adaptability.
- Ownership.
- Stress response.
- Motivation.
- Learning style.

Do not describe behavioral results as diagnosis or personality truth. Use careful language such as “the response suggests” and “based on this answer.”

## Leadership dimensions

- Empathy.
- Accountability.
- Decision-making.
- Communication clarity.
- Conflict handling.
- Team support.

## Communication roleplay dimensions

- Professional tone.
- Clarity.
- Empathy.
- Listening/acknowledgment.
- Solution orientation.
- Pressure handling.

## AI safety checklist

- Use only candidate responses, rubric, and template context.
- Include evidence for every score.
- Avoid unsupported claims.
- Avoid medical, mental health, or protected-class conclusions.
- Never say “hire” or “reject” as an automatic decision.
- Allow human reviewer notes to override or contextualize AI feedback.
