# Evalora Implementation Roadmap

## Month 1: Foundation

Goals:

- Finalize SRS.
- Design database.
- Create UI wireframes.
- Set up repository.
- Build authentication.
- Build role-based dashboard layout.
- Build basic assessment template system.

Deliverables:

- Database schema.
- Auth system.
- Basic dashboard.
- Template creation page.
- Initial API documentation.

## Month 2: Core assessment features

Goals:

- Build candidate session flow.
- Build AI interview chat.
- Build coding assessment page.
- Build non-technical assessment form.
- Store candidate responses.

Deliverables:

- Candidate assessment flow.
- AI chat module.
- Coding module.
- Work-style form module.
- Response storage.

## Month 3: Evaluation and analytics

Goals:

- Build AI evaluation system.
- Build leadership scenario module.
- Build communication roleplay module.
- Build candidate report page.
- Build analytics dashboard.

Deliverables:

- AI scoring system.
- Candidate report.
- Analytics dashboard.
- Interview history.
- Module-level feedback.

## Month 4: Production polish and submission

Goals:

- Improve UI/UX.
- Add error handling.
- Test edge cases.
- Optimize performance.
- Deploy public website.
- Record demo video.
- Complete final PDF report.

Deliverables:

- Public website URL.
- Public GitHub repository.
- Public demo video.
- Final PDF report.
- Testing documentation.
- Individual contribution report.

## MVP priority

### Must have

1. Authentication.
2. Role management.
3. Assessment template creation.
4. Candidate session flow.
5. AI interview chat.
6. Coding assessment.
7. Work-style assessment.
8. AI evaluation.
9. Candidate report.
10. Dashboard.
11. Public deployment.

### Should have

1. Leadership scenario.
2. Communication roleplay.
3. Analytics charts.
4. Report export.
5. Session reconnect handling.
6. Interview history filtering.

### Could have

1. Multiple programming languages.
2. Live cursor tracking.
3. Advanced cheating detection.
4. PDF export.
5. Email invitation.
6. Real-time dashboard updates.
