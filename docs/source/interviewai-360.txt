1. Introduction

1.1 Purpose

This Software Requirements Specification defines the requirements for InterviewAI 360, an AI-powered
interview and assessment platform designed to help organizations evaluate candidates through technical
interviews, coding assessments, behavioral scenarios, leadership tests, communication simulations, and
automated assessment reports.

The purpose of this document is to provide a clear blueprint for the development team and help
stakeholders understand what the system will do, how it will work, and what features will be delivered.

1.2 Project Scope

InterviewAI 360 will allow organizations to create structured interview sessions for different candidate roles.
The system will support both technical and non-technical assessments.

The platform will include:

        · User authentication and role management
        · AI-powered interview chat
        · Live coding assessment
        · Work-style and behavioral assessment forms
        · Leadership scenario testing
        · Communication roleplay simulation
        · Problem-solving and case study assessment
        · Candidate performance scoring
        · AI-generated feedback reports
        · Analytics dashboard
        · Interview history and session management

                                                                           1
The system is designed as a modular assessment platform, meaning new assessment types can be added in
the future without rebuilding the whole system.

1.3 Intended Users

The system is intended for the following users:

User Type       Description

Admin           Manages platform settings, users, assessment templates, and system data

Organization / Client Creates interview campaigns and reviews candidate results

Interviewer / Assessor Manages interviews, reviews AI feedback, and evaluates candidates

Candidate       Completes assigned interviews and assessments

Developer Team  Builds, tests, deploys, and maintains the platform

1.4 Product Goal

The goal of InterviewAI 360 is to reduce the time and effort required to screen candidates while improving
consistency, structure, and insight in the evaluation process.

The platform should not replace human hiring decisions. Instead, it provides organized assessment data,
AI-generated summaries, and performance insights to support human reviewers.

2. Overall Description

2.1 Product Perspective

InterviewAI 360 is a web-based platform that combines interview management, real-time interaction, AI
evaluation, coding assessment, and analytics.

The system will be accessible through a browser and will support multiple user roles. It will include
dashboards for admins and organizations, assessment pages for candidates, and result pages for reviewing
candidate performance.

2.2 Product Functions

The main functions of the system are:

      1. Register and authenticate users
      2. Manage user roles and permissions
      3. Create assessment templates
      4. Create interview sessions
      5. Allow candidates to complete assessments
      6. Provide AI interview conversations
      7. Provide live coding tasks
      8. Support non-technical tests
      9. Generate AI-based evaluations
     10. Display analytics and reports
     11. Store interview history
     12. Export or share candidate reports

2.3 User Classes and Characteristics

Admin

The admin has full control over the platform. Admins can manage users, assessment templates, system
settings, and data.

Organization / Client

The organization user creates assessment campaigns, invites candidates, selects assessment modules, and
reviews candidate reports.

Interviewer / Assessor

The interviewer reviews candidate performance, reads AI-generated summaries, and may manually add
feedback.

Candidate

The candidate joins an assigned interview session and completes the required assessment modules.

2.5 Design and Implementation Constraints

The system must follow these constraints:

        · The website must be publicly accessible for final submission.
        · The GitHub repository must be public.
        · The system must include clear commit history from all team members.
        · The system must include professional documentation.
        · AI-generated results must be explainable and reviewable.
        · The system must not claim to make final hiring decisions automatically.
        · Behavioral assessment must not be presented as medical or mental health diagnosis.
        · Candidate data must be protected and accessible only to authorized users.

18. Final Product Summary

InterviewAI 360 is a web-based AI assessment platform that helps organizations evaluate candidates
through structured technical and behavioral assessments.

The system supports AI interviews, coding tests, work-style assessments, leadership scenarios,
communication roleplay, problem-solving case studies, automated scoring, candidate reports, and analytics
dashboards.

The platform is designed to be modular, scalable, and practical for real-world screening workflows while still
allowing human reviewers to make the final decision.
